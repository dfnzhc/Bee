# Memory

## 1. 概述
内存管理是软件开发中的重点，它对软件的性能以及稳定性都有重大的影响。通常的内存分配与回收策略会有一系列问题，像是频繁的系统调用、内存碎片、内存泄露等问题。
对于 Bee 这个渲染引擎来说，需要管理各种类型的资源数据，如：模型、材质、纹理等。为了提升存取效率、便利性以及系统稳定性，所以一个内存管理系统是必要的。

## 2. 需求

### 稳定

- 内存安全：提供一些边界检查，防止越界访问（Sanitizer？）
- 泄露防护：对内存资源进行监控，能自动回收
- 异常恢复：当出现错误时（如内存不足，资源泄露等），有一定策略保证系统的正常运行

### 性能

- 避免频繁的系统调用开销
- 支持多线程并发分配
- 支持 TLS
- 缓存友好：分配的内存有好的控件局部性


### 功能

- 提供多种分配器：队列、线性、环形、池分配器等
- 跨设备管理：统一 API 抽共享 CPU/GPU 内存（如 CUDA、Vulkan 的设备内存），支持异步的数据交互
- 诊断统计：记录内存峰值、分配热点，并提供可视化报告

### 扩展

- 能够根据运行负载调整分配算法（如在低负载时采用更激进的内存池来提升效率）
- 接入 RenderDoc 来分析内存
